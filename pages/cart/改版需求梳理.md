# 购物车页面改版需求梳理

## 一、需求理解总结

根据您提供的需求说明和示例图片，本次改版主要涉及以下模块的 UI 调整：

1. **收货地址**：样式改为示例中的卡片式布局
2. **店铺行**：右侧按钮根据优惠类型显示「领券」或「优惠」
3. **凑单行**：左侧横向滚动优惠券标签，右侧显示满减阶梯差额及「去凑单」
4. **商品展示**：左右分栏布局，右侧新增标签行（横向滚动）

## 三、修改条目清单

### 条目 1：收货地址样式

**需求**：改为示例卡片样式

**示例结构**（checkout.html 51–65 行）：
- 上方：红色定位图标 + 灰色「收货地址」
- 中间：姓名 + 电话（加粗）
- 下方：详细地址（灰色）
- 右侧：箭头图标

**现有代码位置**：
- 模板：`cart.vue` 约 17–33 行（`.header-address`）
- 样式：`cart.vue` 约 6189–6242 行

**修改方案**：
1. 新增独立 `.address-section` 区块，替代 `.header-address` 中的地址展示
2. 布局：`address-header`（图标+标题） + `address-content`（姓名+电话、详细地址，右侧箭头）
3. 数据：`regionSplic` 需拆为姓名、电话、详细地址；若无则需从 `addressList` 或 `regionData` 中取
4. 标题「采购车」和编辑按钮保留在顶部，但不在地址卡片内
5. 样式参考 `styles.css` 约 822–877 行

---

### 条目 2：店铺行右侧按钮（领券 / 优惠）

**需求**：
- 仅优惠券：显示「领券」
- 既有优惠券又有满减：显示「优惠」

**现有代码位置**：
- 模板：`cart.vue` 约 64–76 行
- 当前逻辑：`storeList.coupuns_num > 0` 时显示 `head-right`，含满减文案和去凑单

**修改方案**：
1. 新增计算属性或方法：`getStoreHeaderBtnText(storeList)`，返回 `'领券'` 或 `'优惠'`
2. 判断逻辑：  
   - 有满减：`storeList.new_list` 中存在 `act_type == 1`（满减）  
   - 有优惠券：`storeList.coupuns_num > 0`  
   - 两者都有 → `'优惠'`，否则 → `'领券'`
3. 模板：右侧按钮改为 `{{ getStoreHeaderBtnText(storeList) }}`，点击事件沿用 `handleCoupon` 或对应跳转
4. 无优惠时可不显示按钮（与示例一致）

---

### 条目 3：凑单行（店铺下方）

**需求**：
- 左侧：有优惠券时，横向滚动展示优惠券标签（满99减10、满199减30 等）
- 右侧：满减下一阶梯「再选 X 立省 Y」+「去凑单」
- 「去凑单」按满减活动维度跳转（act_id）

**现有代码位置**：
- 模板：`cart.vue` 约 78–126 行（`.card-act-goods`）
- 当前：按活动逐条展示，每活动一行

**修改方案**：
1. 新增 `.store-promotion` 区块，结构参考 checkout.html 约 74–84 行
2. 左侧 `.promotion-scroll`：
   - 数据来源：店铺优惠券 + 满减活动，合并为标签数组
   - 使用 `scroll-view scroll-x` 实现横向滚动
   - 标签样式：`tag-manjian`、`tag-coupon` 等（参考 styles.css 约 1018–1030 行）
3. 右侧：
   - 计算下一满减阶梯：`getNextManjianTier(storeList)`，返回 `{ needAmount, saveAmount, actId }`
   - 文案：`再选 {{ needAmount }} 立省 {{ saveAmount }}`
   - 「去凑单」跳转：`/pagesD/cart/coudan/coudan?act_id=' + actId`
4. 赠品行、活动行可保留，但需与新的凑单行区分展示（如赠品行在上，凑单行在下）

---

### 条目 4：商品展示布局（左右分栏 + 标签行）

**需求**（参考图 4）：
- 左侧：仅商品图
- 右侧：
  1. 标题：商品名称
  2. 规格：`规格:xxx`
  3. 新增标签行：横向滚动，支持多种标签（满减、券、7天无理由、限时抢、积分抵扣、黑卡价等）
  4. 价格与数量：券后价、原价、数量选择器

**现有代码位置**：
- 模板：`cart.vue` 约 127–262 行（`.cart-goods-info`）
- 商品结构：复选框 + 商品图 + 标题 + 规格 + 价格 + 数量

**修改方案**：
1. 布局调整：
   - 左侧：`.goods-img` 或 `.order-image`，仅图片，宽度约 80px（160rpx）
   - 右侧：`.order-info` 或 `.cart-goods-content`，垂直排列
2. 右侧结构：
   - 第 1 行：标题（`.goods-title` 或 `h3`）
   - 第 2 行：规格（`.goods-attr` 或 `.order-spec`）
   - 第 3 行：`activity-tags`（新增）横向滚动
   - 第 4 行：价格 + 数量

3. 标签行数据：
   - 字段名建议：`product_tags` 或 `goods_activity_tags`
   - 初期可先用假数据，格式示例：`[{ type: 'manjian', text: '满199减30' }, { type: 'coupon', text: '满99减10券' }, ...]`
   - 后续对接：`item.act_info`、`item.is_membership_card_discount_price` 等

4. 标签样式：
   - 参考 styles.css 约 1018–1030 行：`tag-manjian`、`tag-coupon`、`tag-limit`、`tag-vip`、`tag-points`、`tag-return`
   - 使用 `scroll-view scroll-x` 实现横向滚动

5. 价格与数量：
   - 券后价：`price-label` + `order-price`
   - 原价：`order-original-price`（删除线）
   - 数量：`uni-number-box` 或 `quantity-selector` 样式

### 条目 5：店铺优惠合计行（可选）

**示例**：店铺商品列表下方有「优惠合计 ¥xx.xx 明细」

**现有**：项目中有类似逻辑，但可能不在当前展示区域

**修改方案**：若需与示例一致，可新增 `.store-total` 区块，展示店铺优惠合计及「明细」入口。

## 四、数据字段建议

| 用途 | 建议字段名 | 说明 |
|------|------------|------|
| 商品标签行 | `product_tags` 或 `goods_activity_tags` | 数组，格式 `[{ type, text }]`，type 可选：manjian、coupon、limit、vip、points、return |
| 满减下一阶梯 | 计算属性 | 基于 `storeList.new_list` 中 act_type==1 的活动，计算当前金额与下一阶梯差额 |

## 五、修改顺序建议

1. **条目 1**：收货地址样式（独立模块，影响小）
2. **条目 2**：店铺行按钮文案（逻辑简单）
3. **条目 4**：商品展示布局（核心改动，建议先做）
4. **条目 3**：凑单行（依赖满减计算逻辑）

## 六、注意事项

1. **uni-app 兼容**：示例为 HTML，需改为 `<view>`、`<scroll-view>` 等，并注意 rpx 与 px
2. **多端**：保留 `#ifdef APP-PLUS`、`#ifdef MP-WEIXIN` 等条件编译
3. **国际化**：`$t('lang.xxx')` 需保留
4. **现有功能**：复选框、删除、数量修改、跳转等逻辑不变
